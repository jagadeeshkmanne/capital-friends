8. Google Apps Script Limitations

Capital Friends runs on Google Apps Script, which is a cloud-based scripting platform provided by Google. This platform has inherent limitations that may affect your experience.

8.1 Execution Time Limits

6-Minute Script Timeout: Google Apps Script enforces a maximum execution time of 6 minutes per script run. This means:

  • Complex operations (e.g., calculating portfolio values across hundreds of transactions) may timeout
  • Custom menu functions that process large datasets may fail to complete
  • Email report generation with extensive data may be interrupted
  • You may see error messages like "Script timeout: exceeded maximum execution time"

Workaround: If you encounter timeouts, try reducing the amount of data processed at once or simplifying calculations.

8.2 Email Quotas

Google enforces strict daily email sending limits to prevent spam:

Account Type Daily Email Limit Impact on Capital Friends
Consumer Gmail (free) 100 emails/day Sufficient for daily/weekly reports to 3-5 family members. May fail if you send many emails from other sources.
Google Workspace (paid) 500 emails/day (Business Starter) Handles large families and frequent reports comfortably.

What Counts Toward Your Quota:

  • Each email sent to each recipient counts as 1 email (e.g., sending to 5 family members = 5 emails)
  • Emails sent by ALL your Apps Scripts count toward the same quota
  • Manual emails sent from Gmail also count toward the quota
  • The quota resets at midnight Pacific Time (PT)

⚠️ Quota Exceeded Error: If you hit the email limit, you'll see errors like "Service invoked too many times for one day: email." Email sending will resume automatically the next day. To avoid this, limit the number of recipients or reduce email frequency.

8.3 Spreadsheet API Quotas

Google Apps Script has read/write quotas for accessing Google Sheets:

  • Read Requests: 20,000 calls per day (consumer), 100,000 calls per day (Workspace)
  • Write Requests: 20,000 calls per day (consumer), 100,000 calls per day (Workspace)
  • Simultaneous Executions: Maximum 30 concurrent script executions per user

When This Matters: These quotas are very high and unlikely to be reached with normal Capital Friends usage. However, if you:

  • Run automated triggers very frequently (e.g., hourly updates)
  • Have many other Apps Scripts running simultaneously
  • Process extremely large datasets (10,000+ rows)

...you may encounter "Service invoked too many times" errors.

8.4 Trigger Limits

Google Apps Script limits the number of time-based triggers per script:

  • Maximum Triggers: 20 time-based triggers per user per script
  • Minimum Interval: Hourly (you cannot schedule triggers to run every minute)
  • Trigger Reliability: Triggers may be delayed by up to 15 minutes during high Google Apps Script platform load
  • Trigger Failures: If a trigger fails 10 times in a row, Google may automatically disable it

For Capital Friends: You'll only use 1-2 triggers (daily/weekly email reports), so this limit is not a concern for typical usage.

8.5 Apps Script Environment Limitations

Other technical limitations to be aware of:

  • No External Libraries: Cannot use npm packages or external JavaScript libraries (only Google's built-in services)
  • No Persistent State: Scripts cannot maintain state between executions (data must be stored in sheets)
  • No Direct Database Access: Cannot connect to MySQL, PostgreSQL, or other databases directly
  • Single-Threaded Execution: Scripts run sequentially, not in parallel (no multi-threading)
  • Memory Limits: Large data operations may fail due to memory constraints
  • Network Request Limits: Limited to 20,000 UrlFetch calls per day
